## בניית קובץ EXE

לפני ההתקנה יש לבנות את קובץ ה-EXE מקוד המקור.

> **למה צריך לבנות?**
> הקובץ `red_alert.py` הוא קוד Python — יש להמיר אותו ל-EXE שירוץ על כל מחשב Windows ללא תלות ב-Python מותקן.

### דרישות מקדימות

- **Python 3.9+** מותקן על המחשב
- חיבור לאינטרנט (להורדת ספריות)

### שלב 1 — הרץ את הבנייה

1. פתח את תיקיית הפרויקט
2. לחץ **פעמיים** על `build_exe.bat`
3. אשר הרשאות מנהל אם נדרש

> **מה הסקריפט עושה אוטומטית לפני הבנייה:**
> - סוגר את האפליקציה אם היא פועלת
> - מוחק את התקנה הישנה (`C:\RedAlertIDF`)

### שלב 2 — המתן לסיום

הבנייה לוקחת **2–5 דקות** — תראה הרבה שורות טקסט. זה תקין.

```
[0/5] Closing running instance...
[0/5] Deleting old installation...
[1/5] Installing PyInstaller and dependencies...
[2/5] Running PyInstaller...
      ... (הרבה שורות INFO — נורמלי)
[3/5] Verifying build output...
[4/5] Installing to C:\RedAlertIDF ...
[5/5] Cleaning up temp build files...

=============================================
  Build complete!
  EXE: C:\RedAlertIDF\RedAlertMonitor.exe
=============================================
```

### שלב 3 — סיום

כאשר מופיע **"Build complete!"** — הבנייה הצליחה.
הקובץ `RedAlertMonitor.exe` נמצא כעת בתיקייה `C:\RedAlertIDF\`.

> **שאלה נפוצה — למה תיקייה ולא קובץ יחיד?**
> הרכיב שמציג מפות (WebEngine) דורש קבצי DLL נפרדים שלא ניתן לאחד לקובץ יחיד.
> כל התיקייה `C:\RedAlertIDF\` היא האפליקציה — אל תמחק קבצים ממנה.

### אם הבנייה נכשלת

| שגיאה | פתרון |
|-------|--------|
| `Python not found` | התקן Python מ-python.org ← בחר "Add to PATH" |
| `Build FAILED` | בדוק שיש חיבור אינטרנט ← הרץ שוב |
| האפליקציה לא נסגרה | סגור ידנית מ-Task Manager ← הרץ שוב |

## התקנה

### שלב 1 — הורדה והתקנה

1. פתח את תיקיית הפרויקט
2. לחץ **ימני** על הקובץ `install.bat`
3. בחר **"הפעל כמנהל מערכת"**
4. המתן עד שהתהליך מסתיים (כ-30 שניות)
5. בסוף תישאל **"הפעל עכשיו? (Y/N)"** — הקלד `Y` והקש Enter

> **מה ההתקנה עושה?**
> - מעתיקה את הקבצים ל-`C:\RedAlertIDF\`
> - מגדירה הפעלה אוטומטית עם Windows (Task Scheduler)
> - מפעילה את האפליקציה

## הגדרת הישוב שלך

**מאוד חשוב!** ללא הגדרת ישוב — תקבל התרעות על **כל ישראל**.

### הוספת ישוב

1. לחץ על ⚙ (הגדרות) בווידג'ט

לדוגמה: "בת ים" + "שוהם" — תקבל התרעה כאשר **אחד מהם** מוזהר.

לחץ על ה-**X** ליד שם הישוב ברשימה.

> **טיפ:** אם אתה נוסע לאזור אחר — האפליקציה תזהה את מיקומך החדש ותוסיף את הישוב החדש **זמנית**. הישובים הקבועים שלך יישארו פעילים תמיד.

## מה קורה בהתרעה

### שלב 1 — גילוי ההתרעה

האפליקציה בודקת את שרת פיקוד העורף **כל שנייה**. כשיש התרעה:

### שלב 2 — צליל אזעקה

צליל האזעקה מושמע מיד דרך הרמקולים.

### שלב 3 — מסך מלא

**מסך אדום מלא** נפתח על כל המסכים המחוברים ומציג:

```
┌─────────────────────────────────┐
│  🚀  ירי רקטות וטילים           │
│                                 │
│  בת ים  |  שוהם  |  חולון      │
│                                 │
│  14:23:05    ⏱ 00:45            │
│                                 │
│  [ הבנתי — סגור ]              │
│  [ אני במרחב המוגן ]           │
└─────────────────────────────────┘
```

| כפתור | מה קורה |
|--------|---------|
| **הבנתי — סגור** | סוגר את המסך המלא |
| **אני במרחב המוגן** | עובר לרצועת "מרחב מוגן" (ראה פרק הבא) |

### שלב 4 — חלון מרחף (Overlay)

גם אם סגרת את המסך המלא, מופיע **חלון מרחף קטן** עם ספירה לאחור. הוא נסגר אוטומטית לאחר הזמן שהוגדר בהגדרות.

### שלב 5 — מפה

מפת ישראל נפתחת ומסמנת באדום את הישובים תחת התרעה.
המפה **נסגרת אוטומטית לאחר 10 שניות**.

### שלב 6 — הודעת Telegram

אם הגדרת Telegram — תגיע הודעה לטלגפון תוך שנייה.

### שלב 7 — עדכוני חדשות (60 שניות לאחר מכן)

לאחר 60 שניות מההתרעה, המערכת מתחילה לחפש ידיעות חדשות רלוונטיות (ראה פרק [עדכוני חדשות](#עדכוני-חדשות-לאחר-התרעה)).

## המרחב המוגן

### כניסה למרחב מוגן

לחץ **"אני במרחב המוגן"** במסך המלא — תופיע **רצועה אדומה** בתחתית המסך:

```
🔴  יש להישאר במרחב המוגן  |  00:08:30 נותרו
```

הרצועה מציגה ספירה לאחור לפי הנחיות פיקוד העורף.

### שחרור פיקוד העורף

כאשר פיקוד העורף מכריז "מותר לצאת", הרצועה **עוברת לירוק**:

```
🟢  ניתן לצאת — פיקוד העורף שחרר
```

הרצועה נסגרת אוטומטית לאחר 8 שניות.

## הגדרת Telegram

קבל התרעות ישירות לטלפון דרך בוט Telegram.

### יצירת בוט Telegram (פעם אחת)

1. פתח Telegram ושלח הודעה ל-**@BotFather**
2. הקלד `/newbot`
3. תן שם לבוט (לדוגמה: "Red Alert שלי")
4. תקבל **Bot Token** — שמור אותו

### הגדרה באפליקציה

1. לחץ ⚙ הגדרות ← **"📱 הגדרות טלגרם"**
2. הדבק את ה-**Bot Token** בשדה המתאים
3. סמן ✅ **"הפעל Telegram"**
4. שלח הודעה כלשהי לבוט שלך מהטלפון
5. לחץ **"🔍 זהה אוטומטית"** — ימלא את ה-Chat ID אוטומטית
6. לחץ **"📨 שלח הודעת בדיקה"** — אם הגיעה הודעה לטלגרם, הכל עובד!
7. לחץ **שמור**

### הגדרה אוטומטית של הבוט (אופציונלי)

לחץ **"🔧 הגדר בוט אוטומטית"** — מגדיר שם, תיאור ופקודות לבוט שלך בלחיצה אחת.

### בדיקה מקיפה

לחץ **"🧪 בדיקה מקיפה"** — יפתח מסך בדיקה מלא (כאילו הייתה התרעה אמיתית) וישלח הודעה לטלגרם. מציג דוח: מה עובד ומה לא.

### פורמט ההודעה בהתרעה

```
🚀  ירי רקטות וטילים
🏙  בת ים  |  שוהם
🕐  14:23:05
```

## עדכוני חדשות לאחר התרעה

לאחר התרעת **רקטות או כלי טיס**, האפליקציה מחפשת אוטומטית ידיעות חדשות:

### תהליך החיפוש

```
התרעה התקבלה
      ↓
המתנה של 60 שניות
      ↓
חיפוש ב-RSS: ynet + walla + mako
      ↓
בדיקה כל 30 שניות (עד 10 דקות)
      ↓
נמצאה ידיעה רלוונטית? → נשלחת לטלגרם מיד
      ↓
אחרי 10 דקות ← הודעת סיכום
```

### מה נחשב "ידיעה רלוונטית"?

ידיעה רלוונטית חייבת לכלול:
- **שם הישוב** שקיבל את ההתרעה
- **מילת מפתח** כגון: נפילה, ירוט, פגיעה, כיפת ברזל, שיגור, רקטה, טיל...

### פורמט הודעת חדשות בטלגרם

```
📰  עדכון חדשות — ירי רקטות וטילים
🏙  בת ים  |  שוהם
🕐  14:23:05

• ירוט מוצלח של רקטה מעל בת ים
  🔗 https://www.ynet.co.il/...

• פגיעה בשטח פתוח בשוהם
  🔗 https://rss.walla.co.il/...
```

> **אם לא נמצאו ידיעות** — תגיע הודעת טלגרם: "לא נמצאו עדכוני חדשות רלוונטיים"

## הגדרות מתקדמות

לחץ ⚙ בווידג'ט לפתיחת הגדרות.

### הגדרות עיקריות

| הגדרה | מה לעשות |
|-------|----------|
| **📍 בחר ישובים** | הוסף את הישוב שלך |
| **🔊 צליל** | הפעל/כבה + בחר סוג צליל |

```json
  "cities": ["בת ים", "שוהם"],
  "cat":    "1",
  "ts":     "2026-03-19T14:23:05"
}
```

1. **האם Telegram מופעל?** — הגדרות ← 📱 Telegram ← ודא שה-✅ "הפעל Telegram" מסומן
2. **האם ה-Chat ID מוגדר?** — שלח הודעה לבוט ← לחץ "🔍 זהה אוטומטית"
3. **בדיקה:** לחץ "📨 שלח הודעת בדיקה" — אם לא הגיעה הודעה, ה-Token שגוי
4. **בדיקה מקיפה:** לחץ "🧪 בדיקה מקיפה" לדוח מפורט

## שאלות נפוצות

**ש: האפליקציה אוכלת הרבה זיכרון?**
ת: כ-80-150MB. הרמה הזו נורמלית עבור אפליקציית PyQt5 עם WebEngine.

**ש: האם ההתרעות מגיעות בזמן אמת?**
ת: האפליקציה בודקת כל **שנייה**. ייתכן איחור של 0-1 שניות ביחס לשרת פיקוד העורף.

**ש: האם הנתונים עוברים דרך שרת חיצוני?**
ת: לא. האפליקציה פונה **ישירות** לשרת פיקוד העורף (oref.org.il). לא עוברים נתונים דרך שרת ביניים.

**ש: מה ה-"cat" בהיסטוריה?**
ת: קטגוריית ההתרעה: 1=רקטות, 2=כלי טיס, 3=רעידת אדמה, 4=חומרים מסוכנים, 5=מחבלים, 6=צונאמי.

**ש: איך מעדכנים לגרסה חדשה?**
ת: הורד את הגרסה החדשה, הרץ `build_exe.bat` (סוגר ומוחק את הישנה אוטומטית) ואחר כך `install.bat`.
